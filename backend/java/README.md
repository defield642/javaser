# Gexup Java Backend

Render-friendly Spring Boot service.

Run locally:

```bash
./mvnw spring-boot:run
```

If you do not have `mvnw`, use your local Maven install:

```bash
mvn spring-boot:run
```

Routes:
- `GET /health`
- `GET /ping`
- `GET /servers`
- `POST /optimize`
- `POST /tunnel/config`
- `GET /relay/health`
- WebSocket `/relay/socket`

Deploy on Render as a Java service with:
- Build command: `mvn package`
- Start command: `java -jar target/gexup-java-backend-0.1.0.jar`

Render env vars:
- `PORT=8080`
- `RELAY_TOKEN` generated by Render
- `RELAY_CONNECT_TIMEOUT_MS=2000`

The backend derives its public relay host and port from the incoming Render request, so you do not need to hardcode your `.onrender.com` domain in the Java code.
